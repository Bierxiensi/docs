# 计算机系统基础

为什么说《计算机系统基础》让你收益匪浅？你从中学到了什么？ 答：在本科学习阶段学习的《数据结构》、《操作系统》、《计算机网络》和《计算机组成原理》等基础课程中虽然都有很多详尽的知识，但整体学完后还是无法将这些知识串联起来。在学习这门课程后，特别是工作后站在web程序员的角度上，可以清晰的了解到所编写的应用在计算机上的执行过程。

比如，在本机浏览器打开web应用时：

  1. 首先会与DNS服务器建立UDP连接进行域名解析得到网站的IP地址。本机会将目标IP地址、目标端口号等信息封装进入TCP报文段，通过三次握手与客户端服务器建立TCP连接，这时就可以访问应用；
  2. 在应用运行时，浏览器会为应用创建几个进程并分配内存空间，其中网络进程负责前述请求的建立、维护与网络资源加载。 而渲染进程会负责程序的执行以及渲染等工作。为了更好地实现多道程序并发执行，渲染进程引入了多线程分别进行页面渲染，代码执行，事件处理；
  3. 由于web应用使用javascript编写，是解释型语言，代码执行线程中的解释器会逐行将高级语言翻译成机器指令并由CPU立即执行；
  4. 为了解决CPU执行指令和访存速度不匹配，主存容量限制问题。现代计算机多采用cache和虚拟存储技术。在这样的系统里，当CPU在执行指令的访存过程中，首先会查询当前快表中是否有当前指令所在的页的页表项，如果有则从页表项中取出改页的虚拟地址并转换为物理地址，并用物理地址去cpu的cache中获取该块信息并执行。在不发生缺页与未命中的情况下，这样就完成了该条指令的执行。

死锁产生条件：互斥、不剥夺、请求和保持、循环等待

# 为什么说《计算机系统基础》让你收益匪浅？你从中学到了什么？

答：本科阶段基础课程知识虽然详尽，但学完后无法将这些知识串联起来。学习这门课后对计算机知识有了系统性的理解。 比如打开一个web页面，域名解析成IP地址的阶段使用到了UDP协议与DNS协议，页面信息的传输依赖于TCP协议，是计算机网络的知识。 页面的展示和运行依赖于浏览器开启的渲染、事件处理进程，是操作系统的知识。Js是解释型语言，在代码执行时解释器会直接解释成机器指令由CPU执行，是编译原理和组成原理的知识。当CPU执行指令和访存速度不匹配时使用了cache和虚拟存储技术，这是操作系统的知识。 以上就是我理解的web应用在计算机上的执行过程，我感到受益很多。

# 可以简单讲一下你的毕业设计吗？

答：我的毕业设计课题是《基于Python的气象数据可视化分析》，包括数据预处理、数据分析、数据可视化三个部分。 其中数据预处理阶段主要对是对获取的数据进行清洗、切片、存储，使用到了python的数据处理库pandas等，最终目标数据保存成.cvs文件。 数据分析阶段通过对不同日期区间，不同气象特征数据进行观察来预测未来的走势，使用到了线性回归与RNN模型。其中线性回归用于预测单组的气象特征，RNN用于预测多组气象特征。 数据可视化阶段，主要对原始数据和数据分析阶段过程中产生的数据进行可视化展示，使用到了Echarts、百度地图以及python的flask框架。 通过毕业设计，提高了理论联系实际解决问题的能力，以及阅读论文的能力。

# 你利用线性回归和RNN都做了什么？

线性回归（Linear regression）：利用最小二乘函数对单组气温数据进行预测，容易陷入局部最优解的求解中。 RNN：考虑通过多组特征值来预测气温变化，RNN擅长处理预测的序列数据。包含输入层、隐藏层、输出层，隐藏层上一次的值作为这一次的输入的权重，计算完毕将输出传递给前馈层，最终返回预测结果。 数据处理：通过三维点阵分布选择一段时期内关联性强的多组特征值;进行无量纲化转换成规格一致的数据;通过区间缩放与二值化处理得到输入数据。

由前述复试经历，可总结几点如下：

1，老师打断较多，且不知道什么时候会打断，所以问题的要点一定要在前面说处理来，如还未打断再逐条解释各个要点；

2，个人政治陈述和英文自我介绍最好能控制在1分钟之内，不然打断风险极高，开个头迅速进入个人亮点部分；

3，回答还是力求直接切入正题。我复试时废话比较多。可能是工作中经常给领导汇报而养成的习惯，回答问题喜欢先铺设背景、达成双方共识。在复试中，就没有这个必要，老师问什么，就直接了当切入问题点而回答，因为老师都是领域专家，双方已经有基础共识了。老师问这个，考生回答半天别的，就容易被认为知识不够或耍滑头；

4，复试也考验运气。我运气偏好，复试时抽到的题都不难，唯一的难题就是银行家算法。后来我看了看那个算法，要真搞清楚这个算法还是需要花费一些时间的；

5，非全在专业课上，编译原理不要碰；其它科目，找一找总结性的笔记，看看概念即可。非全要工作，就须讲究复试效率，看看概念，不至于冷场，就能过。不同于非全，全日制还是需要做好万全准备。
